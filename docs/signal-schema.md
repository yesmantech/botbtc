# Signal JSON Schema

## MT5 → Bridge Signal Payload

The MT5 EA sends JSON newline-delimited messages over TCP:

```json
{
  "signal_id": "SIG-20260521-000001-000001",
  "strategy_id": "btc_micro_momentum_v1",
  "symbol": "BTCUSD",
  "side": "BUY",
  "signal_price": 107500.50,
  "bid": 107500.00,
  "ask": 107501.00,
  "spread": 1.00,
  "velocity_250ms": 12.50,
  "velocity_500ms": 8.30,
  "velocity_1000ms": 5.10,
  "acceleration": 7.40,
  "ema_fast": 107498.75,
  "ema_slow": 107495.20,
  "micro_atr_1s": 15.30,
  "micro_atr_2s": 22.10,
  "legacy_lots": 0.01,
  "contract_size": 1.0,
  "t0_tick_ms": 1760000000000,
  "t1_signal_ms": 1760000000002
}
```

### Field Descriptions

| Field | Type | Description |
|-------|------|-------------|
| `signal_id` | string | Unique signal identifier (generated by MT5) |
| `strategy_id` | string | Strategy name/version |
| `symbol` | string | MT5 symbol name |
| `side` | string | `"BUY"` (long) or `"SELL"` (short) |
| `signal_price` | float | Mid price at signal generation |
| `bid` | float | Best bid at signal time |
| `ask` | float | Best ask at signal time |
| `spread` | float | ask - bid |
| `velocity_250ms` | float | Price change over last 250ms (USD) |
| `velocity_500ms` | float | Price change over last 500ms (USD) |
| `velocity_1000ms` | float | Price change over last 1000ms (USD) |
| `acceleration` | float | velocity_250ms - velocity_1000ms |
| `ema_fast` | float | Fast micro EMA value |
| `ema_slow` | float | Slow micro EMA value |
| `micro_atr_1s` | float | Average true range over 1 second |
| `micro_atr_2s` | float | Average true range over 2 seconds |
| `legacy_lots` | float | MT5 lot size (for reference) |
| `contract_size` | float | MT5 contract size (for reference) |
| `t0_tick_ms` | int64 | Tick arrival timestamp (ms since epoch) |
| `t1_signal_ms` | int64 | Signal generation timestamp (ms since epoch) |

## Bridge → MT5 ACK Response

```json
{
  "type": "ACK",
  "signal_id": "SIG-20260521-000001-000001",
  "received_ms": 1760000000005,
  "status": "accepted"
}
```

### ACK Status Values

| Status | Meaning |
|--------|---------|
| `accepted` | Signal received and queued for processing |
| `rejected_risk` | Rejected by risk engine |
| `rejected_stale` | Signal too old |
| `rejected_duplicate` | Duplicate signal_id |
| `error` | Parse or processing error |

## Protocol

- Transport: TCP socket on `127.0.0.1:PORT`
- Encoding: UTF-8 JSON
- Framing: newline-delimited (`\n`)
- Direction: bidirectional (signal → ACK)
- Connection: persistent, reconnect on failure
